# Promotion Playbook

Match tactics to the repo's stage. Repo quality comes first — promotion multiplies whatever impression the repo makes, good or bad.

## Stage 0 → ~100 stars: Launch
- **Show HN** (news.ycombinator.com): title format "Show HN: {name} – {what it does}". Post morning US time, weekday. First comment from the author explaining motivation. One realistic HN or Reddit hit can be worth 500–2000 stars in a day.
- **Reddit**: r/programming, language/framework subreddits, r/opensource, r/selfhosted (if applicable), r/webdev "Showoff Saturday". Read each sub's self-promotion rules; write as a story ("I built X because Y"), not an ad.
- **Product Hunt**: works for developer tools with a visual angle; prepare gallery images/GIF.
- **Direct network**: personal post on X/LinkedIn; relevant Discord/Slack communities (only where on-topic).
- Launch when the repo passes the audit — a launch spike onto a weak README is wasted.

## Stage ~100 → ~1k stars: Content flywheel
- **Technical blog posts** (Dev.to, Hashnode, Medium, own blog): "how it works" internals posts and "how to solve {problem} with {tool}" tutorials. Cross-post; each post links to the repo.
- **Awesome lists**: PR the repo into relevant `awesome-*` lists — steady, compounding traffic.
- **Comparison content**: "{name} vs {popular alternative}" captures evaluators.
- **Answer questions** on Stack Overflow / Reddit where the tool solves the asker's problem (disclose you're the author).
- **Short video/GIF demos** on X — visual demos travel far in dev social media.

## Stage 1k+: Community & momentum
- **Milestone posts** (1k, 5k, 10k stars): "lessons learned" retrospectives perform well on HN/Dev.to and drive a second wave.
- **Release notes as content**: every meaningful release gets a short announcement post/tweet with a visual.
- **Grow contributors**: good-first-issues, fast PR reviews, credit contributors publicly — contributors bring their networks.
- **Talks/podcasts**: meetups and conference lightning talks; niche dev podcasts.
- **GitHub Trending**: concentrated bursts of stars (launch/content hits) can put a repo on trending, which compounds. You can't game it reliably; you can time announcements to stack on the same day.

## What to avoid
- Star-for-star exchanges, purchased stars, giveaway campaigns for stars — detectable (dead profiles, spike patterns), ban-risk, and they destroy credibility with the developers who matter.
- Spamming communities with pure links; every post needs standalone value.
- Launching the same thing to the same audience repeatedly without news.
